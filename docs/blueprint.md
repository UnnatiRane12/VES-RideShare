# **App Name**: VES RideShare

## Core Features:

- User Authentication: Allow users to log in using their institute (VES) ID and verify their identity.
- Dashboard: Display a personalized dashboard greeting the user and offering options to create or enter a sharing.
- Create Sharing Room: Provide a form-like page for users to input sharing details such as room name, auto status, destination, and starting point.
- Smart Matching: Match the users to a similar or on the way destination via AI tools and algorithm which detects common routes or nearby destinations. Also incorporates user preferences like maximum travel time.
- Room Details: Display details of each sharing room, including the owner's identity and participants.
- Room Management: Automatically remove sharing rooms from the listing when the passenger limit is reached.
- Expiration and Editing: Allow users to set an expiration time for their sharing rooms and enable editing of room details after creation.

## Style Guidelines:

- Primary color: Deep blue (#3F51B5) for a trustworthy and professional feel, reflecting the institution's brand.
- Background color: Very light gray (#F5F5F5), providing a clean and neutral backdrop.
- Accent color: Teal (#009688), used sparingly for interactive elements and key actions to draw attention.
- Body and headline font: 'Inter', a sans-serif font, will ensure the application has a machined, neutral aesthetic.
- Use clean, modern icons to represent various features such as destination points, car status, and user profiles.
- Maintain a grid-based layout to ensure the application is accessible on multiple device formats.
- Implement subtle transitions and animations when users create a new room or joins an existing sharing room to show smooth flow.
